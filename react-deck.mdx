class Button extends React.Component {
    constructor(props) {
        super(props) // Call React.Component constuctor
        this.state = { color: "green" } // Define initial state
        this.isClicked = this.isClicked.bind(this) //Required for methods
    }

        if (this.state.color === "green") this.setState({color: "red"})

    render() {
        return <ButtonWrapper>

                color={this.state.color}
                onClick={this.isClicked}

                // Uses the buttontext variable passed in from parent

class Button extends React.Component {
   constructor(props) {
        super(props) // Call React.Component constuctor
        this.state = { color: "green" } // Define initial state
        this.isClicked = this.isClicked.bind(this) //Required for methods
        this.updateCount = 0
    }
    isClicked() {
        if (this.state.color === "green") this.setState({color: "red"})
        else this.setState({color: "green"}) this.setState({color: "green"})
    }
    componentDidUpdate() {
        this.updateCount++
    }
    render() {
        return <ButtonWrapper>
            Child Component
            <StyledButton color={this.state.color} onClick={this.isClicked}> {this.props.buttontext} </StyledButton>
            Number of updates: {this.updateCount}

    }
}

## Things to be aware of

- JSX is not _exactly_ HTML
- - HTML: class JSX: className
- - Intersperce HTML with javascript logic
- setState:
- - Not the same as setting a variable! Triggers rerender of component
- - Don't use setState() inside render() (why?)

## List of potential topics
- Event handlers are difficult and verbose
- - Two way data bindings
- The DOM is very intensive—any manipulation to it tells the browser to perform a change
- - Using a Virtual DOM, we can alleviate this
- - Virtual DOM objects correspond to DOM objects, but dont trigger changes unless React tells it to [^2]
- Organizing code in a big javascript file sucks
- - Component architecture vs other paradigms
- Writing a server is a lot of work
- - Utilization of browser API-s like history to encroach on server-side functionality
- - React Router
- - Can focus on only writing API endpoints in your backend!

## Little gotchas
- setState()
- - Not the same as just changing the the class's state variable! Also tells React that a change occured and to re-render the element!
- Async function
- - Callbacks, Promises, async/await
- Composition > Inheritance
- Lifting up state
- JSX is not HTML, more similar to functional Javascript
- - {`<p hi="yeet">"yeeeet2"</p> <=> React.createElement("p", {hi: "yeet"}, "yeeeet2")`}
- - {`<p hi="yeet"><p>"nested"</p></p> <=> React.createElement("p", {hi: "yeet"}, React.createElement("p", null, "nested"))`}
